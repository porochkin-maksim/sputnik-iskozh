# PHP Core Codex – Универсальные правила и стандарты

> **Целевая версия PHP: 8.4**  
> Весь код должен использовать синтаксис и возможности PHP 8.4 там, где это уместно, и быть совместимым с 8.3 как
> минимальной версией.

## 1. Базовые требования к файлу

- Каждый PHP-файл **обязан** начинаться с `declare(strict_types=1);`.
- Открывающий тег `<?php` – без пробелов после него.
- Закрывающий тег `?>` **не используется** (кроме чистых шаблонов).
- Кодировка: UTF-8 без BOM.

## 1.1. Обязательность локальных правил

- Правила из `.codex/rules/*.md` обязательны к исполнению и не могут игнорироваться молча.
- Если задача затрагивает рефакторинг, архитектуру домена, controller orchestration или application-слой, сначала нужно свериться с релевантными правилами из `.codex/rules/*`, а затем следовать им.
- Если реализация временно отклоняется от правила, это нужно явно указать пользователю до внесения изменений и объяснить причину отклонения.

## 2. Стандарты кодирования (PSR-12 + улучшения)

- Отступы: **4 пробела** (не табуляция).
- Фигурные скобки для классов, методов и функций: **на новой строке**.
- Фигурные скобки для управляющих конструкций (`if`, `for`, `while`): **на той же строке**.
- Максимальная длина строки: 120 символов (рекомендуется 80-100).
- После запятых в списках аргументов и элементов массива – пробел.
- Битовые операторы, операторы сравнения и присваивания обособляются пробелами.
- Все внешние классы, интерфейсы, enum и исключения должны подключаться через `use`.
- Не оставляй неиспользуемых подключений в `use`.
- Не используй fully-qualified class names в телах методов, свойствах, сигнатурах и выражениях, если это можно выразить через `use`.
- Не пиши `\Exception`, `\RuntimeException`, `\ReflectionClass` и подобные имена прямо в коде метода; подключай их через `use`.
- Если объект создаётся и сразу участвует во fluent-chain, предпочитай проектный синтаксис без лишних скобок:
  ```php
  $item = new SomeEntity()->setId($id);
  ```
- Не добавляй оборачивающие скобки вокруг `new ...()` перед `->`, если без них код валиден и читаем.
- Не дублируй одну и ту же логику преобразования данных между слоями. Если mapping между domain и persistence уже вынесен в mapper, повторять его в factory или repository запрещено.

## 3. Именование

| Элемент                    | Стиль               | Пример                       |
|----------------------------|---------------------|------------------------------|
| Классы, интерфейсы         | PascalCase          | `InvoiceService`             |
| Методы, функции            | camelCase           | `getTotalAmount()`           |
| Свойства (включая private) | camelCase           | `$invoiceId`                 |
| Константы классов          | UPPER_SNAKE_CASE    | `STATUS_PAID`                |
| Интерфейсы                 | Суффикс `Interface` | `InvoiceRepositoryInterface` |
| Трейты                     | Суффикс `Trait`     | `LoggableTrait`              |
| Enum                       | PascalCase          | `InvoiceStatus`              |
| Переменные                 | camelCase           | `$totalAmount`               |

## 4. Типизация и строгость

- **Всегда** объявляй типы параметров и возвращаемых значений.
- Для `void` – указывай явно.
- Для `mixed` – используй только когда тип действительно неизвестен (редко).
- Используй union типы (`int|string`) и intersection типы (`A&B`).
- Используй `readonly` классы и свойства (PHP 8.2+).
- Используй конструктор property promotion.
- В доменах `core/Domains/*` отдельный `DTO`-слой не нужен.
- Если доменная модель данных нужна как объект, она должна называться `*Entity`.
- Не создавай параллельно `Entity` и `DTO`, если это не разные по смыслу модели и пользователь явно не требует оба слоя.

## 5. Возможности PHP 8.3 и 8.4 (обязательные к применению)

### PHP 8.3

- Константы в трейтах.
- Типизированные константы (`public const string STATUS = 'paid';`).
- Атрибут `#[\Override]` для методов, переопределяющих родительские.
- Динамические обращения к классам: `ClassName::CONST` синтаксис унифицирован.
- Функции `json_validate()` (если нужно).

### PHP 8.4 (предпочтительно)

- **Property hooks** – вместо ручных getter/setter, когда нужна логика:
  ```php
  public string $name {
      get => $this->name;
      set => $this->name = strtoupper($value);
  }
